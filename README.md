# My Kitchen Stock (AI献立提案 兼 食材在庫管理システム)

**思考コスト削減を目的とした、在庫連動型AI献立提案システムです。**
**毎日の「今日何作ろう…」という悩みをゼロに。冷蔵庫の余り物からAIが最適な献立を自動提案します。**

前職（調達・購買部門）で培ったプロフェッショナルな在庫管理ロジックと、生成AI（Gemini API）を掛け合わせ、家庭のフードロスと献立の悩みを同時に解決します。

## 導入による提供価値（ユーザーインパクト）

ターゲット層：毎朝のお弁当作りに追われる、忙しい共働き世代・子育て中の主婦

- **思考コストの完全排除:** 冷蔵庫の前でフリーズしてしまう献立考案のストレスを、AIとの連携により解放します。
- **経済的コスト削減とSDGsへの貢献:** 期限切れ前の食材を的確に消費させることで、毎月の給食費相当（約400円/日）のコスト削減を実現しつつ、家庭内のフードロス・ゼロに貢献します。

## 開発背景：「2児の母のリアルな悩み」×「購買業務の知見」

職業訓練校のチーム開発にてレシピアプリを作成した際、「データベース設計の甘さからくる在庫連動のエラー」や「UIの統一感の欠如」という課題に直面しました。
また、私自身が2児の母として毎日キッチンに立つ中で、「在庫管理以上に、毎日ゼロから献立を考えること自体が一番しんどい」と痛感していました。

そこで、「実運用で破綻しない強固なDB設計」を最優先としつつ、「一切思考せずに済むAI献立生成」を掛け合わせた完全版として、本システムを個人開発でフルリメイクしました。

## こだわりの設計（技術的アプローチ・開発スコープ）

本アプリは、要件定義・DB設計・API連携・UI実装まで一貫して個人で開発しました。特に、データ整合性を担保するためのDB設計と、AI出力の不確実性を前提としたAPI設計に注力しています。

### 1. AI出力の不確実性を制御する「エラーハンドリング」

生成AI（Gemini API）特有の「出力揺れ」によるシステム不整合を防ぐため、プロンプト設計とレスポンス制御を徹底しました。

- **構造化データの強制:** AIの出力を自然言語ではなく厳密なJSON形式に指定し、システム側で安全にパース・保存できるフローを構築。
- **レスポンスのバリデーション処理:** AIの返答エラーやJSON構文エラーをキャッチするだけでなく、不正なデータの保存を防止する処理を実装し、システムダウンを防ぎます。

### 2. データの信頼性を優先した「引き算の設計」

当初は「レシピ作成後、使った食材を在庫から自動減算する機能」を構想していましたが、AIの出力揺れによる在庫連動エラーのリスクを考慮し、あえて実装を見送りました。機能の多さよりも「ユーザーに誤った情報を提示しないこと（データの信頼性）」を優先した設計判断です。

### 3. 購買業務の知見を応用した「プロフェッショナル在庫管理」

- **ステータスによる厳格な分離:** 冷蔵・冷凍の状態をDB上で明確に分離。
- **動的な期限管理（賞味期限アラート）:** Carbonを用いた現在時刻との相対計算により、「今すぐ使い切るべき食材」をダッシュボード上で赤・黄のアラートとして視覚的に警告します。

### 4. 視覚的ノイズを排除したUIと「栄養・彩りの可視化」

忙しい朝でも直感的に操作できるよう、アースカラーを基調とした「大人シンプル」なUIを徹底。Bladeの動的レンダリングに加え、**栄養素データをフロント側で集計し、グラフとして可視化（JavaScript）する**ことで、直感的なUIを実現しました。

## 使用技術と選定理由

- **Laravel 10.x:** MVC構造による責務分離と、将来的な機能拡張や保守性を意識した設計のため。
- **Google Gemini API (gemini-2.5-flash):** 高速なレスポンスタイムを活かし、ユーザーを待たせないリアルタイムな献立生成を実現するため。
- **MySQL (JawsDB):** 在庫管理や中間テーブル等、構造化データの厳密な整合性担保のため。
- **Blade / Tailwind CSS:** コンポーネント指向を取り入れたシンプルな構成でUI構築を高速化し、UX改善に注力するため。
- **JavaScript (Ajax / Chart.js):** 非同期通信とフロントエンドでのデータ集計・可視化により、操作ストレスのないUXを実現するため。

## 技術的なポイントまとめ

- データ整合性を意識したDB設計（在庫ロジックの厳格なステータス管理）
- AI出力の構造化とバリデーション制御（システムエラーの未然防止）
- UXを優先した機能設計（データの信頼性を守るための「引き算」の意思決定）
- フロントエンドでの栄養素データ集計とグラフ可視化
